#+title: manager-implementer
#+orgasmic_version: 1

* Convention: manager-implementer (single worker)

Use when one agent is both manager and implementer: the prompt asks for
direct implementation in this session, or no worker runtime or dispatch
surface is available. Direct-edits are for source code only; `.orgasmic/`
state stays daemon-mediated. This mode trades the staged pipeline for speed:
iterate fast, reconcile later.

** Skip while iterating
- Creating or claiming a task heading before editing.
- Per-edit lifecycle stage transitions and pre-flight dispatch tx.
- The dedicated REVIEW stage and reviewer dispatch.

** Keep while iterating
- `.orgasmic/*.org`: CLI-only (`orgasmic decision create`, `orgasmic
  architecture create`, `orgasmic node body set|append` — see `orgasmic
  --help`).
- Code markers: single-line `// orgasmic:<node-id>` or `# orgasmic:<node-id>`
  comments. Add or refresh one only when it materially helps future
  reconciliation; never let a marker widen the scope you edit.
- `gotchas.org`: read before source edits; add an entry when surprised.

** Reconcile later
At a natural pause (the burst lands, the session ends, or a handoff):
- Close or update affected task headings via `orgasmic task update`.
- Refresh `tasks/handoff.org` via CLI when the session carries manager
  continuity state.
- One batched `orgasmic tx record` summarizing the burst is enough.
