#+title: no skill installed
#+orgasmic_version: 1

* Convention: manager fallback without the `/orgasmic` skill

Use when:
- `entry.org` routes you to Manager
- no `/orgasmic` skill exists at `~/.agents/skills/orgasmic/SKILL.md`
- no visible skill registry exposes `/orgasmic`

** Role
- You are Manager.
- Do not act as Worker unless the prompt is a strict worker dispatch.

** Read

1. `project.org`: identity, mission, constraints, and verification notes.
2. `tasks/goal.org`: goal only.
3. `tasks/handoff.org`: continuity, liveness.
4. `tasks/*.org` state files (`backlog.org` … `cancelled.org`), recent `tx/*.org`: task state.
5. `git status --short`, `git log --oneline -20`, `git worktree list`: local
   work state.

** Report
- current goal
- done so far
- in flight
- next planned action
- divergence

** Act
- Bare `/orgasmic`, `/orgasmic recall`, or a status/pickup request: brief and
  stop.
- `/orgasmic resume` or "continue": brief; continue only with clear next action
  and no divergence.
- New arbitrary request: treat as manager direction.

** Write
- `.orgasmic/*.org` state: install or start the runtime (`/orgasmic install`,
  `orgasmic daemon start`); do not hand-edit. Read-only orientation stays
  allowed.
- Product source: do not edit as Manager.
- Worker work: dispatch or hand off when supported.
- Same-agent Manager+Worker work: follow `conventions/manager-implementer.org`.
