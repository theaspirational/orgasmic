- Verification: confirm `orgasmic` resolves in a fresh PowerShell and cmd session.

Note: WSL and Git-Bash/MSYS2 already work via the unix path; this task is specifically native PowerShell/cmd.

* BACKLOG TASK-567JG Respawn dispatch completion watcher on boot reattach (no last.txt after daemon restart)    :daemon:dispatch:reattach:
:PROPERTIES:
:ID: TASK-567JG
:SEVERITY: medium
:END:

Distinct second failure mode discovered while fixing TASK-B05AM (the stall-clobber, fixed in ed6a6fc). When the daemon restarts MID-RUN, the run is reattached (boot reconciliation, api::reattach_live_runs_on_boot) but the dispatch completion watcher — a tokio task spawned only at dispatch time (api.rs spawn_dispatch_completion_watcher, called only from the dispatch handler ~3453) — died with the old daemon process. No watcher is respawned for the reattached run, so when it later releases, write_dispatch_completion_artifacts is never called and NEITHER last.txt NOR stdout.log is written. Observed on TASK-Y2ZQJ (session had a 'reattach' lifecycle event; 15053 assistant chunks + run_complete present, yet no last.txt).

Root cause: the session's RunMeta lifecycle event carries transport/harness/project_id/worktree/driver_config but NOT last_path/stdout_path (the dispatch artifact paths, derived CLI-side from the brief filename stem). So on reattach the daemon cannot reconstruct where to write the artifacts.

Fix direction: persist last_path + stdout_path (and worktree_path) in the RunMeta lifecycle event at dispatch time; on boot reattach (reattach_live_runs_on_boot), reconstruct a DispatchCompletion from RunMeta and call spawn_dispatch_completion_watcher for each reattached worker/reviewer run. Add a test: reattach a run with RunMeta artifact paths, release it, assert last.txt is written. Until then, the harness transcript remains the rescue source for restart-mid-run dispatches.

Workaround already in effect: prefer in-process agent reviewers over rmux dispatches while the report path is fragile; rescue from harness transcript when a dispatch loses its report.
