* BACKLOG TASK-41B0T Design semantic progress detection for worker runs    :design:
:PROPERTIES:
:ID: TASK-41B0T
:PRIORITY: medium
:RELATES_TO: TASK-2D0EJ
:END:

TASK-2D0EJ deliberately leaves supervisor timeout semantics unchanged. Design a provider-agnostic runtime contract that separates transport activity (ACP heartbeats and tool poll events) from useful worker progress, so a stale yielded command token cannot keep a run healthy indefinitely. Preserve explicit finalize, cancellation, recovery authority, and existing timeout compatibility. Specify observables, configuration/migration, escalation ownership, and deterministic regressions for live heartbeats plus a vanished subprocess before implementation.
